# 战斗能力阶段（Stat Stages）— 设计定稿

本文汇总 **ChampionTale** 战斗中「能力阶段」与物/特/变化技能、等级、HP 等规则，供实现与改数值时对照。与当前工程里「平面 `*_mod` 加减」方案不同，**统一改为阶段倍率**（速度相关技能效果亦改为改**速度阶段**）。

---

## 1. 能力阶段适用范围

| 能力 | 是否有阶段 | 说明 |
|------|------------|------|
| 攻击 | 是 | 阶段 −6～+6 |
| 防御 | 是 | 同上 |
| 特攻 | 是 | 同上 |
| 特防 | 是 | 同上 |
| 速度 | 是 | 同上；原技能中 `self_speed_delta` / `target_speed_delta` 等**平面速度修正**改为**速度阶段**变化 |
| **HP** | **否** | 不参与能力阶段；见下文「HP 与等级」 |

阶段变化在应用到数值前 **clamp** 到 **[−6, +6]**。

---

## 2. 倍率公式（Gen 6+）

设阶段为整数 \(n \in [-6, 6]\)，倍率 `mult(n)` 为：

- **\(n \ge 0\)**：`mult = (2 + n) / 2`
- **\(n < 0\)**：`mult = 2 / (2 - n)`

\(n = 0\) 时倍率为 1。

对参与阶段的能力，**有效能力**（用于伤害、排序、tooltip 等）建议为：

1. 先算 **基础值**（含种族基础与**等级**等**非阶段**加法，见第 4 节）。
2. 再乘 **`mult(对应阶段)`**。
3. 再 **`max(1, floor(…))`** 得到整数（或项目统一取整规则）。

**HP**：不使用阶段倍率；满血与当前 HP 见第 4 节。

---

## 3. 结算顺序（与特性钩子）

建议顺序：

1. 取各项 **基础值（含等级）**。
2. 对攻/防/特攻/特防/速度乘 **能力阶段倍率**，并取整、下限 1（速度至少 1 等同理）。
3. 伤害计算再套用 **威力阶段 / 粗伤阶段** 等**特性钩子**（如 Bully、迅疾等），避免与「能力阶段」重复定义同一层。

---

## 4. 等级与 HP

### 4.1 等级参与的能力

**攻击、防御、速度、特攻、特防** 在运行时与 UI（如 tooltip）中展示的数值，均需体现 **等级** 的影响（具体加法或公式在实现层与既有 `BattleUnitDefinition` / `BattleUnitRuntime` 对齐，并保持五维规则一致）。

### 4.2 HP（无阶段）

- **不设 HP 能力阶段**。
- **`BattleUnitDefinition.hp_max`** 表示：该单位在资源 **`level` 字段所写等级**下的 **满 HP**。
- 战斗开始时，运行时用该满 HP 初始化当前 HP（除非另有机制改血）；若将来存在「运行时等级与资源 `level` 不一致」，需另订缩放规则（当前原型可暂不展开）。

---

## 5. 技能分类：物理 / 特殊 / 变化

- 每个技能需有 **分类枚举**（命名可为 Physical / Special / Status 等）。
- **物理**：伤害结算取 **攻击** 与目标 **防御**（经各自阶段后的有效值）。
- **特殊**：伤害结算取 **特攻** 与目标 **特防**。
- **变化（Status）**：用于不造成伤害或主要为场地/状态效果的技能；**不参与**物/特攻防选取；可与现有 `deals_damage` 等字段组合约束（例如在 `deals_damage == false` 时常为变化类）。

实现时需在 `SkillData`（或等价资源）上增加该字段，并为现有技能资源补全分类。

---

## 6. 与旧「平面修正」的关系

- **`atk_mod` / `def_mod` / `spd_mod` 不再使用**，统一改为 **能力阶段**。
- **特性示例 — 力量代价**：每回合结束时 **攻击阶段 +1**、**防御阶段 −1**（可叠，仍受 ±6 限制）。
- **技能对速度的影响**：原 **整数速度修正** 改为 **速度阶段** 的增减（具体每技能改几阶段由数据配置，与旧手感可对齐后调整）。

---

## 7. UI / Tooltip

- Tooltip（及同类战斗内展示）应显示：
  - 经 **等级与阶段** 后的 **有效能力值**（HP 仅等级与资源定义下的满血/当前血，**无阶段**）。
  - 攻、防、特攻、特防、速度当前所处的 **阶段**（如 +1、−2、0）。

---

## 8. 数据与运行时补充说明

- **`BattleUnitDefinition`** 已支持 **特攻/特防种族值**（`spatk_base` / `spdef_base`，小于 0 时可表示「与攻击/防御同值」等规则，与图鉴一致）。
- **`BattleUnitRuntime`** 需维护五项阶段变量，并实现上述 `effective_*` 计算；伤害公式在 **`BattleSkillResolver`**（或等价模块）中按技能分类选取攻防属性。

---

## 9. 导出与资源

图鉴、遭遇等 **不依赖** 本节的阶段逻辑即可独立存在；战斗导出仍遵循工程内 `preload` / 资源引用约定（参见 `docx/export-pokedex-troubleshooting.md`）。
